Vue.js basics(part2)

  1. @mouseover,@mouseleave,@dblclick,@mousemove
    当创建如 handleEvent(){}方法时,若()中不传入自定义参数，则唯一的参数 e 默认为事件参数，可以输出 e.type 展示为事件类型；若有自定义参数，则 e.type（变成 undefined）不能展示出事件对象，此时若希望事件对象被传递：在调用时用参数$event
    （VUe.js 中说，允许向方法传入自定义参数以代替原生事件；若需要在内联事件处理器中访问原生 DOM 事件，可以向该处理器方法传入一个特殊的 $event 变量）
    e.offsetX,e.offsetY：框中的光标位置 offset 指光标相对于左上角的偏移量
  2. outputting lists
    v-for:遍历数据数组 <ul><li v-for=”xxx in array”></li></ul>,使用时调用某属性时可以用 xxx.jpg 等
    用 v-for 时需要绑定 key，使每个元素的 key 都是独一无二的
  3. attibute binding
    v-bind(:)：用于动态绑定属性，如<a v-bind:href=”xxx”>，还可以用于 img src
  4. dynamic classes :class=”{fav:book.isFav}”
  5. toggling favs

  6. computed properties
    定义依赖于其他数据属性的数据，当其他数据更新时，computed 里的数据也会更新
    如：computed{filter(){return this.xxx.filter((xx)=>xx.)}}
    （xxx 为数组名，xx 为里面的数值名）

  7. 通过 prop 把父组件里的数据传入子数据：
    在 template 的标签属性里放入<xxx header=”header” text=”text”>等数据，数据可以在 data 中定义然后通过:(v-bind)绑定。在子组件中用<script>的 export default{}引用 props:[“header”]，最后在子组件 template 里引用

    • <style scoped>,则样式只用于当前组件
    • 把给 style 赋值的属性写得更具体，如在标签名前面加类名
    • 全局样式：在 assets 中添加 css 文件，在 main.js 中使用 imort”…”把文件引入

  9. emitting custom events 自定义事件（从子组件触发，让父组件收到）：
    定义的时候：this.$emit(“随机取的变量名”,要传入的data数值等(this.xxx,此数值名要传入自定义方法名的()里作为参数)
    引用的时候：@变量名=”定义的方法名”，()里写上data数值名
    在子组件中 methods 里定义事件，事件名照常使用，{}中用 this.$emit(“xxx”)从而让父组件给 xxx 赋具体值：父组件用 props 传给子组件的标签里加上<@xxx=”某父组件中方法”>

  10. click event modifiers 事件修饰符
    在@click 右边加.right:只有点击鼠标右键时起作用。
    .shift：得同时按下 shift 键,有此功能的还有.ctrl，.alt，.meta
    @click.self:只有点该元素时奏效， 点该元素里的子元素也不行

  11. slots 插槽
    传递 templates 这种可重复用的组件，不传递简单数据(string/boolean/array)
    用法：root 页面中不用自闭合组件标签，而把它打开：<xxx>传递内容</xxx>。再在被传递组件中插入<slot></slot>，这里面的是默认 slot 的默认内容

    • 具名插槽：
      在 root 中 slot 模版中的任意位置插入一个模版如<template>,再<template>中给模版命名如 v-slot:links,插入内容后在对应组件中需要插入具名插槽的标签对里插入<slot name=””></slot>。
    • 默认模版：可以在要传入的组件中<slot></slot>中间插入某些值，当没有要传入的默认数据时就会显示出来。

  12. creating another modal
    组件用很强的可复用性 reusable，复用过程中只需要改变 root 中的传递内容，不需要改组件文件里的东西（modal.vue

  13. using teleport 传送门
    在 index.html 中新建一个 div，给其一个 class 或者 ID，
    再使 vue 文件中的某些方法单独与此 div 绑定：将 app.vue 中对应的标签名改成<teleport to=””(.xxx/#xxx)>等。
    再将全局 css（root 文件中的 style）里#app 对应的样式前加上.xxx/#xxx 使其可以与全局样式绑定

forms&inputs

  1. v-model:将 data 与表单 form 里的值绑定（input,select，checkboxes…)
    双向数据绑定：

    • 从输入到组件数据的数据绑定：当数据在表单中随着输入更新，它绑定到组建的 data 也更新，
    • 从 data 到 template：当更新 data 中的值或者用 method 等方法更新组件中数据时，输入字段中的内容也会随之更新
  • 对于 select 来讲，指定默认值可以使 v-model 绑定的数据与 value 相等
  • 对于 checkbox 复选框来讲，如果只有一个框，指定 v-model 的默认值是 boolean。如果是有相同 v-model 的多个输入框，要给每个 input 设置一个 value 来标记，指定的默认值是一个数组，当哪个框是 checked 时其对应的 value 也进入数组
  1. 可以单独设置某个类别的 input 元素样式，如：input[type=’checkbox’]{}
  2. keyboard events
    v-for 里面必须为每个元素添加一个独一无二的 key，
    [].includes(‘x’)：判断某数组中是否包含某元素并返回 true/false
    str.includes(“xxx”):查找字符串是不是包含某字符
    当用 keyup.alt 来获取事件时，与 alt 键一同按下的键可以被读取但不会再显示在 input 中
  3. .prevent:阻止默认的行为（如加载页面）用法如：@submit.prevent。可以代替 event.preventDefault()
  4. (x)=>{return xxx }可以写成(x) => xxx
  5. form 表单中 button 标签默认的类型是 submit，即默认的 button 点击就会触发表单的提交事件，触发页面的自动刷新。

the vue router

前端路由是指 Hash 地址与组件之间的对应关系。一个 web 网站只有一个唯一的 HTML 页面，不同组件的切换要依赖于前端路由来完成。

  1. 路由链接比普通的锚点链接快很多，因为它只是改变组件而不是向服务器发送新请求且会拦截这些请求。
  2. 可快捷地改变点中的导航栏中路由条目的样式。
  3. 可将数据与根目录下的 <router-link> 标签绑定，在 to 属性前加：且不直接硬编码它的路径，而是输入一个代表这条路径的对象，给出 name 和对应的值如：{name:’About’}

folder structure

  1. 路径 path 里参数用冒号隔开 如 /jobs/:id .在 url 中遵循此命名规则的可以直接刷新到对应窗口。
  2. 输出动态值可以使用$，route信息里包括当前路线的任何信息如路线参数。如：当path里的参数变成id时，用$route.params.id 可以实时获取 id 名字
    可以在 data 里给变量赋值为对应的 route 参数，输出时可以直接引用
    制作可以传递路由参数的动态路由链接：
    <router-link>中用:to={}绑定 name 和 params 属性，可以用 params 属性将参数传递到 url，params 里面的值是一个对象，该对象中每个键都是一个 route 参数， 对象里的属性可以是 id 并给它赋值。
    可以任意路由参数链接到组件的 props 上，比如在其他文件里定义的 id 值可以在文件里用 props 接收，此时组件里就不需要在 data 里返回对应的 id 等参数了
    使用方法：在 index.js 里将对应组件路由设置 props:true，在对应组件里引入 props:[‘id’]

redirects&404’s
redirects:在 index.js 中设置，让路由中旧地址也跳转到新地址{path:’旧地址’,redirect:’新地址’}
为任意不存在的路由设置一个 catch-all 路由-显示 404 页面：1.在 views 下创建一个 vue 页面 2.在 index.js 下创建对象，path 属性的值是‘/：catchAll(.*)’ 2.在对象里为 name 和 component 赋值，component 的值是刚刚创建的页面 3.在页面上端 import xxx(name) from xxxx

Programmatic Navigation
在 methods 中获取$router（勿混淆：router 用来移动它们，route 用来获取当前路由里的信息），有 go()方法，若用 go(-1)意味着倒退一个页面，go(1)则是前进一个页面。
若在 methods用 redirect 方法，前面依旧需要加this.$router.用 push()表示推进一个页面，()里面输入和 router-link 标签里要输入的{}path/name 一样的对象，如 name:”about”

fetching data

  1. 从 vue 组件内部获取外部数据，用 json server 的包模拟向其发出请求的 api
    用法： 1. 建一个 data 文件夹，在里面生成一个 xxx.json 文件，文件像一个对象那样以花括号开头，里面存放的键需要用双引号引起来。若对象里的值是字符串，必须用双引号引起来（单引号’’的话则不算数）2. 在此项目的本地服务器安装 json server 包：npm install （-g） json-server 3. 用其读取文件：json-server –watch 文件夹名/文件名

  2. 在 mounted hook 里用 fetch（）请求 1.异步 回调函数 将获取的响应作为参数。res.json 是使用 fetch api 的方式 ，这个同样是异步函数，返回一个 promise

the composition

  • composition API 1.可以在 setup 函数里将逻辑都整合在一起 2.便于创建可复用的逻辑功能
    如在 setup(){}中，可以定义数据、方法，不需要再用 data 和 methods 操作。
    $refs 不能在 setup 中使用，因为此时该组件还没被创建出来
  • refs:
    使用时必须添加语句：import {ref} from ‘vue’
    用法：
    1. 引用模版中的值：在 setup 中使用时，需要把它赋值给变量，且在一开始没运行到 return 的时候，ref 依旧等于默认值而不是绑定的元素的值。有一些 ref 可以用的功能如变量名为 p 时，有 p.value.classList.add(),p.value.textContent=’’等

  1. using refs for reactivity 创建反应变量
    在默认情况下，composition API 中定义的变量不是反应变量，当更改里面的数值时，模版中的值不会自动更新（与 data 相反）。因此需要在 setup 函数中创建反应变量。方法：1. 用 refs 创建反应变量：让变量=的右边用 ref()环绕。return 变量且在模版中直接引用。setup 函数中得到它的值的语法是：name.value

    ref 的值可以是对象，数组，等等。当 ref 值是对象时，调用时需要用 xxx.xx 的格式调用某具体属性，在方法中使用或重新赋值时要用 xxx.value.xx 的格式。

    @click=’’里面不一定要加方法名，还可以加 js 语言比如 xx++

  • reactive
    reactive 和 ref 最主要的区别：ref 必须用 value 属性去使用里面的值而 reactive 不需要用 value 属性。
    缺点：1. 不能使用原始值，如赋以一个变量 reactive 值时，不能是 string 这种 primitive type 原始类型，否则它不能成为反应变量。
    2.ref 在创建外部组件函数时能更好地工作，因为它们暴露在外部函数时也能保留反应性
    所以通常还是使用 refs

  • using computed values
    使用的时候和 ref 一样需要 import 导入
    给变量赋于 computed 值时，xxx=computed(()=>{return …}) xxx 等于{}里面 return 的值，且需要最后在 return 中返回 xxx，调用时直接在 template 中调用 xxx 即可。
    除了简单返回一个值，computed 经常是根据其他使用过的值来返回一个过滤过的新值。
    filter() 方法可以对数组进行过滤，创建一个新的数组，新数组中的元素是通过检查指定数组中符合条件的所有元素。xxx= xx.filter((item)=>{…..})
    它的特点和以前一样可以定义依赖于其他数据属性的数据，随着其他数据一起更新

  • watch&watchEffect

    1. 使用前需要 import 导入
    2. watch()的第一个参数是输入需要观察的变量，第二个是函数定义的方法，每次第一个变量值改变时都会触发
    3. watchEffect(()=>{})在组件第一次加载的时候会运行一次，当{}里输入变量的 value 时，它会在变量变化时运行且显示出变量的值
    4. 如何阻止监视：直接将该方法存储在变量 xxx 中，阻止的时候调用。在阻止的方法里面写上 xxx()

  • 在 composition API 中使用 props:
    按照上面的写法创建变量的前提下，创建一个标签，标签名和要导入的子组件名相等，在该标签里：绑定变量名和值 xxx=”xxx”。import 导入组件名和详细路径，在 export default 的 components:{}中也要加上组件名。
    接收 props 的组件中用法不变，直接在组件内部用 props:[‘xxx’]。接收的界面里若要在 setup(){}中使用 props，需要在括号里加上 props 参数，然后在里面使用 props.xxx。

    在 component API 如 setup(){}里用 hooks 时需要 import，此外写法开头需要加 on,如 onMount
